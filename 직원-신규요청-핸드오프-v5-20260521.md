## 🟠 P1 — 핵심 기능 불능

### Q1. 진척보고서 저장 → 활동탭 파일에 안 쌓임
- **사장님 원문**: "진척보고서가 저장이되면 옆에 활동탭에 파일에 쌓여야돼"
- **현재 동작**: 진척보고서를 저장해도 활동탭 `파일` 섹션에 안 보임.
- **코드 단서**: `project-slide-over.tsx` L676~864의 `ActivityTab` 은 두 소스를 합쳐서 표시 — 
  (1) `data.documents`(documents 테이블 deal_id 매칭) 
  (2) `signedFiles`(`quote_approvals` stage in {estimate/contract/progress_report/completion/settlement} 의 우리 서명본 PDF, `/contracts/signed/[id]` 링크).
  코드 상 **stage='progress_report' 행이 quote_approvals에 있으면 자동으로 보이게** 되어 있음. → 안 보이는 이유는 **저장 시 quote_approvals INSERT 또는 documents INSERT 어느 쪽에도 안 들어가는 것**으로 추정.
- **라우팅**: docs-agent(서명/문서 저장 흐름) + growth-agent(progress_report_stage_card 저장 트리거). 
  관련 파일 후보: `src/components/progress-report-stage-card.tsx`, `src/lib/quote-approvals.ts`, `src/lib/project-rules.ts`, deal stage='in_progress' 폼.
- **할 것**: 진척보고서 "저장" 액션의 INSERT 경로 확인 → `quote_approvals`(stage='progress_report', deal_id) 또는 `documents`(deal_id, type='progress_report')에 row가 실제 생성되는지 prod 데이터로 재현. 안 들어가면 INSERT 누락분 보강. 들어가도 활동탭 쿼리가 안 잡으면 쿼리/매핑 정정.
- **완료 정의**: 진척보고서 저장 → 활동탭 `파일` 섹션 즉시(또는 invalidate 후) 새 항목 표시. 재현 캡처 첨부.

### Q2. 활동탭 파일에 계약서·견적서 여전히 안 들어옴
- **사장님 원문**: "활동탭 파일에 계약서랑 견적서 여전히 저장안되고"
- **현재 동작**: 계약서·견적서 저장 후에도 활동탭 파일 섹션에 안 보임.
- **코드 단서**: Q1과 동일 — `signedFiles`는 `quote_approvals.our_signed_at || decided_at || sent_at` 시점이 있어야 표시. **저장만 하고 발송/서명 안 한 단계**라면 우리 서명·발송·결정 timestamp가 비어 있어 파일 목록에서 누락 가능. 직원 의도는 "저장 즉시 파일에 누적"인데 현재 코드는 발송/서명 이벤트 발생 후에만 표시.
- **라우팅**: docs-agent + growth-agent. 
  관련 파일: `src/components/project-quote-stages.tsx`, `src/lib/quote-approvals.ts`, 견적서·계약서 저장 폼.
- **할 것**: 
  1. `quote_approvals` 행이 stage='estimate'/'contract'에 대해 저장 시점에 실제 생성되는지 prod 데이터로 확인.
  2. 생성된다면 → `signedFiles` 빌드 조건을 "저장만 됐어도 표시"로 완화 (현재는 timestamp 우선 → "저장 시점/생성 시점"으로 변경, or `created_at` fallback).
  3. 생성 안 된다면 → 저장 액션에 quote_approvals INSERT(또는 documents INSERT) 추가.
  4. 통합: Q1·Q2는 같은 근본 원인일 가능성 — 한 번에 해소.
- **하지 말 것**: BLOCKED 영역(홈택스/현금영수증 codef 분기) 접근 금지. 기존 발송/서명 흐름 회귀 금지.
- **완료 정의**: 견적서·계약서·진척보고서 **저장 직후**(발송·서명 전이라도) 활동탭 파일에 표시. 각 항목 stage 아이콘(📋견적/📝계약/📊진척) + 클릭 시 해당 상세로 이동. 재현 캡처 3건.

## 🟡 P2 — 신규 기능

### Q3. 활동탭의 담당자 선택(추가/제거) UI
- **사장님 원문**: "활동탭의 담당자 선택도 가능하게 해줘"
- **현재 동작**: 활동탭 `담당자 (N)` 섹션은 **읽기 전용 목록**만(L792~819). 기존 담당자 추가/제거 UI는 1638c6a로 `/deals` 페이지에 있고, 새 프로젝트 슬라이드 패널에는 없음.
- **라우팅**: growth-agent — `src/components/project-slide-over.tsx` ActivityTab의 담당자 Section.
- **할 것**: 활동탭 담당자 섹션에 (a) "+ 담당자 추가" 버튼 → 사용자 검색 자동완성(같은 회사 활성 사용자) + 역할 선택(담당/검토/참여) → INSERT into deal_assignments. (b) 각 담당자 행 우측에 ✕ 제거 버튼 → soft remove(`is_active=false`) 또는 DELETE. 기존 1638c6a /deals 페이지의 UI 패턴 재사용 권장. RPC `get_my_assigned_deals` 의미 영향 없음(직원 위젯은 활성만 카운트).
- **하지 말 것**: 직원 권한에서 마음대로 본인을 매니저로 추가하는 것 금지(권한 체크 — 매니저/팀장/admin만 추가·제거 가능). RLS 준수.
- **완료 정의**: 활동탭에서 담당자 추가/제거 가능, 변경 즉시 목록 invalidate. 96da17a의 담당자 지정 알림 트리거 재사용으로 알림 자동 발송.
