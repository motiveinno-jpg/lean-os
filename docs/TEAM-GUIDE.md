# 오너뷰(OwnerView) 실무자 협업 가이드

> 이 문서는 **회계담당자**와 **사업본부장**이 오너뷰 서비스를 실서비스로 만드는 과정에서 어떻게 참여하는지 안내합니다.

---

## 1. 우리가 하는 일

오너뷰는 중소기업 올인원 경영 OS입니다. 현재 코드는 완성되어 있지만, 서버 설정이 안 되어 있어서 실제로 작동하지 않습니다. 이것을 **2개월 안에 실서비스로 만들어 4,000개 기업에 배포**하는 것이 목표입니다.

### 역할 분담

| 역할 | 누가 | 하는 일 |
|------|------|---------|
| **개발** | Claude (AI) | 코드 수정, 버그 수정, 기능 구현, 배포 |
| **회계/세무 QA** | 회계담당자 | 세금계산서, 급여, 거래내역 등 숫자 정확성 테스트 |
| **비즈니스 QA** | 사업본부장 | 딜 관리, 대시보드, 전체 UX 테스트 + 서비스 가입 |
| **최종 승인** | 대표님 | 주요 방향 결정, 런칭 승인 |

## 2. 사업본부장 — 즉시 해야 할 일 (Day 1)

아래 서비스들에 가입하고, 발급된 키/정보를 대표님 또는 Claude에게 전달해주세요.

### 2-1. Vercel 가입 + GitHub 연동 (5분)

1. https://vercel.com 접속
2. "Sign Up" → **"Continue with GitHub"** 클릭
3. GitHub 계정: `motiveinno-jpg` 으로 로그인
4. Vercel 대시보드에서 "Import Project" → `lean-os` 저장소 선택
5. **완료 후 전달**: Vercel 프로젝트 URL

### 2-2. Google OAuth 설정 (10분)

1. https://console.cloud.google.com 접속 (회사 Google 계정)
2. 좌측 메뉴 → "API 및 서비스" → "사용자 인증 정보"
3. "사용자 인증 정보 만들기" → "OAuth 클라이언트 ID"
4. 애플리케이션 유형: **웹 애플리케이션**
5. 이름: `OwnerView`
6. 승인된 리다이렉트 URI 추가:
   - `https://njbvdkuvtdtkxyylwngn.supabase.co/auth/v1/callback`
7. **완료 후 전달**: 클라이언트 ID + 클라이언트 보안 비밀번호

### 2-3. Kakao OAuth 설정 (10분)

1. https://developers.kakao.com 접속 → 로그인
2. "내 애플리케이션" → "애플리케이션 추가하기"
3. 앱 이름: `OwnerView`, 사업자명: `모티브이노베이션`
4. 생성 후 → "앱 키" 메뉴에서 **REST API 키** 복사
5. "카카오 로그인" → 활성화 설정 **ON**
6. "Redirect URI" 추가:
   - `https://njbvdkuvtdtkxyylwngn.supabase.co/auth/v1/callback`
7. **완료 후 전달**: REST API 키

### 2-4. CODEF API 신청 (15분, 발급 3~7영업일)

1. https://codef.io 접속 → 회원가입
2. "API 사용 신청" 클릭
3. 필요 서류: 사업자등록증, 대표자 신분증 사본
4. 용도: "자사 서비스 은행 거래내역 연동"
5. **발급 후 전달**: Client ID + Client Secret

### 2-5. 공공데이터포털 API 신청 (5분, 즉시 발급)

1. https://data.go.kr 접속 → 회원가입
2. 검색: "세금계산서" → "국세청 세금계산서 발급 내역 조회 서비스"
3. "활용신청" 클릭 → 사유 입력
4. **발급 후 전달**: API 인증키

### 2-6. 토스페이먼츠 가맹점 등록 (Week 2에 진행)

> CODEF가 우선이므로 2주차에 진행합니다.

1. https://developers.tosspayments.com 접속
2. "가맹점 등록" → 사업자 정보 입력
3. 서류: 사업자등록증, 통장사본
4. **테스트 키**는 즉시 발급됨 → 먼저 전달
5. **운영 키**는 심사 완료 후 (3~5영업일)

## 3. 버그/이슈 보고하는 방법

### GitHub Issues 사용법

1. https://github.com/motiveinno-jpg/lean-os/issues 접속
2. "New Issue" 클릭
3. 아래 양식으로 작성:

```
제목: [화면이름] 문제 한줄 설명

## 어디서 발생했나요?
- 페이지: (예: 대시보드, 딜 관리, 세금계산서 등)
- URL: (브라우저 주소창의 주소)

## 무엇이 문제인가요?
(구체적으로 설명. 예: "매출 합계가 실제와 100만원 차이남")

## 재현 방법
1. (첫 번째 단계)
2. (두 번째 단계)
3. (문제가 발생하는 단계)

## 스크린샷
(반드시 첨부! 캡처 후 이슈 작성란에 붙여넣기하면 자동 업로드됩니다)

## 심각도
- [ ] 🔴 긴급 (서비스 사용 불가)
- [ ] 🟡 중요 (기능은 되지만 잘못된 결과)
- [ ] 🟢 개선 (불편하지만 사용 가능)
```

### 스크린샷 찍는 법
- **Mac**: `Cmd + Shift + 4` → 영역 드래그
- **Windows**: `Win + Shift + S`
- **iPhone**: 전원 + 볼륨 업 동시 누르기

### 라벨 붙이기 (선택)
이슈 작성 후 오른쪽에서 라벨을 선택할 수 있습니다:
- `bug` — 오류, 잘못된 동작
- `ux` — 사용하기 불편한 점
- `feature` — 새로운 기능 요청
- `critical` — 긴급 수정 필요

## 4. QA 테스트 가이드

### Phase 2 (Week 2~3): 핵심 기능 테스트

서비스가 작동하기 시작하면, 아래 시나리오를 하나씩 테스트해주세요.

#### 회계담당자 테스트 목록

| # | 테스트 | 확인 사항 | 통과? |
|---|--------|----------|-------|
| 1 | 세금계산서 수동 입력 | 공급가액, 세액 자동 계산 정확한지 | |
| 2 | 세금계산서 5건 입력 후 매칭 | 거래내역과 자동 매칭되는지 | |
| 3 | 거래내역 Excel 업로드 | 파일 인식, 자동 분류 정확한지 | |
| 4 | 직원 급여 계산 | 국민연금 4.5%, 건보 3.545% 등 정확한지 | |
| 5 | 경비 청구 → 승인 | 청구서 작성 → 승인자에게 표시되는지 | |
| 6 | 대시보드 매출 합계 | 입력한 데이터 합계와 대시보드 수치 일치하는지 | |
| 7 | 미수금 표시 | 발행했지만 입금 안 된 건이 정확히 표시되는지 | |

#### 사업본부장 테스트 목록

| # | 테스트 | 확인 사항 | 통과? |
|---|--------|----------|-------|
| 1 | 회원가입 (이메일) | 가입 → 인증메일 → 로그인 성공하는지 | |
| 2 | 회원가입 (Google) | Google 계정으로 바로 로그인되는지 | |
| 3 | 회원가입 (Kakao) | 카카오 계정으로 바로 로그인되는지 | |
| 4 | 온보딩 위저드 | 회사 정보 입력 → 완료까지 막힘 없는지 | |
| 5 | 딜 생성 | 새 딜 생성 → 스테이지 변경 → 저장되는지 | |
| 6 | 딜에 문서 첨부 | 견적서/계약서 파일 업로드 되는지 | |
| 7 | 거래처 등록 | 신규 거래처 추가 → 딜과 연결되는지 | |
| 8 | 팀원 초대 | 이메일로 초대 → 수락 → 로그인 되는지 | |
| 9 | 대시보드 6개 위젯 | 모두 숫자가 표시되는지 (빈칸 없는지) | |
| 10 | 모바일 접속 | 핸드폰에서 주요 화면 사용 가능한지 | |

### 테스트 결과 보고

- **통과**: 해당 항목에 체크 (별도 이슈 불필요)
- **실패**: GitHub Issues에 버그 등록 (위의 양식대로)
- **애매함**: 이슈에 `question` 라벨 붙여서 등록

## 6. 자주 묻는 질문

**Q: GitHub 사용법을 모르겠어요**
A: Issues 탭에서 "New Issue" 버튼만 누르면 됩니다. 글 쓰고 스크린샷 붙이면 끝입니다.

**Q: 테스트 중 서비스가 안 돼요**
A: 개발 중이므로 일시적으로 안 될 수 있습니다. 30분 후 다시 시도해보시고, 계속 안 되면 이슈를 등록해주세요.

**Q: API 키를 어디에 전달하나요?**
A: 대표님에게 직접 전달하거나, 슬랙/카톡으로 보내주세요. 절대 GitHub 이슈나 공개 채널에 올리지 마세요.

**Q: 긴급한 문제가 발생했어요**
A: GitHub Issue에 `critical` 라벨을 붙이고, 대표님에게 바로 연락해주세요.
