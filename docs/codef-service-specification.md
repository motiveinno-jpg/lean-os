## 1. 서비스 개요

### 1.1 서비스 소개

**OwnerView(오너뷰)**는 중소기업 및 스타트업을 위한 **올인원 경영관리 SaaS**입니다.

재무/세무, 인사/급여, 문서/계약, 프로젝트 관리, 협업 채팅을 하나의 플랫폼에 통합하여, 별도의 ERP나 회계사무소 없이도 대표가 실시간으로 회사 현황을 파악하고 의사결정할 수 있는 환경을 제공합니다.

### 1.2 서비스 목적

| 항목 | 내용 |
|------|------|
| **핵심 가치** | 중소기업 대표의 경영 가시성 확보 + 반복 업무 자동화 |
| **타겟 시장** | 직원 3~50명 규모의 중소기업/스타트업 |

### 2.2 데이터 활용 목적

#### 은행 입출금 내역 (P0)
- **목적**: 거래내역 자동 수집 → 자동 분류 → 대시보드 지표 계산
- **사용 기능**:
  - CEO 대시보드 6대 생존지표 (현금잔고, 런웨이, 월 번율, 미수금 등)
  - 거래내역 자동 분류 (거래처별, 비용유형별)
  - 3-Way 매칭 엔진 (거래내역 ↔ 세금계산서 ↔ 주문서 대조)
  - 현금흐름 예측 (30일/90일)
- **조회 주기**: 1일 1회 배치 동기화 (사용자가 수동 트리거도 가능)
- **데이터 보관**: Supabase PostgreSQL (AES-256 암호화, 회사별 RLS 격리)

#### 은행 계좌 잔액 (P0)
- **목적**: 실시간 자금 현황 파악
- **사용 기능**:
  - 대시보드 현금 잔고 위젯
  - 런웨이 계산 (현금 / 월 번율)
  - 결제 큐 실행 전 잔액 확인
- **조회 주기**: 대시보드 접속 시 실시간 조회 (캐싱 5분)

#### 법인카드 승인/청구 내역 (P0)
- **목적**: 카드 지출 자동 수집 → 비용 관리
- **사용 기능**:
  - 법인카드 지출 자동 분류 (15개 비용 카테고리)
  - 법인카드 절세 자동 분류 (접대비/복리후생비/교육훈련비 등)
  - 카드 청구금 예측 → 결제 큐 자동 등록
  - 부가세 매입세액 공제 자동 계산
- **조회 주기**: 1일 1회 배치

## 3. 기술 아키텍처

### 3.1 시스템 구성

| 구성 요소 | 기술 스택 | 역할 |
|----------|----------|------|
| **프론트엔드** | Next.js 16 + React 19 + TypeScript 5.9 + TailwindCSS 4 | SPA/SSR 웹 애플리케이션 |
| **백엔드/DB** | Supabase (PostgreSQL 15 + PostgREST + Auth + Realtime) | API + DB + 인증 + 실시간 |
| **Edge Functions** | Deno Runtime (Supabase Edge Functions) | CODEF API 호출, 데이터 변환, 이메일 발송 |
| **호스팅** | Vercel (SSR + CDN) | 프론트엔드 호스팅, 자동 배포 |
| **도메인** | www.owner-view.com | 프로덕션 서비스 도메인 |
| **결제** | Stripe Live | 구독 과금 |
| **모니터링** | Sentry | 에러 추적 + 성능 모니터링 |
| **이메일** | Resend Pro | 트랜잭션 이메일 |

### 3.2 CODEF 연동 아키텍처

```
[사용자 브라우저]
      │
      ▼
[Next.js App (Vercel SSR)]
      │
      ▼ (authenticated request)
[Supabase Edge Function: codef-sync]
      │
      ├── 1. company_settings에서 CODEF 인증정보 조회
      ├── 2. CODEF OAuth 토큰 발급 (Client Credentials)
      ├── 3. CODEF API 호출 (connectedId 기반)
      ├── 4. 응답 데이터 파싱 + 변환
      ├── 5. transactions 테이블에 upsert (external_id로 중복 방지)
      └── 6. 자동 분류 엔진 실행
              │
              ▼
      [Supabase PostgreSQL]
      ├── transactions (거래내역)
      ├── bank_accounts (계좌 잔액 업데이트)
      └── company_settings (CODEF 설정 저장)
```

### 3.3 Edge Function 상세 (codef-sync)

이미 구현 완료된 Edge Function입니다:

- **파일**: `supabase/functions/codef-sync/index.ts`
- **인증**: Supabase service_role 키 + JWT 검증
- **기능**:
  - OAuth 2.0 Client Credentials로 CODEF Access Token 발급
  - connectedId 기반 은행 거래내역 조회
  - connectedId 기반 카드 승인내역 조회
  - 응답 데이터 → OwnerView transactions 테이블 형식 변환
  - external_id 기반 중복 방지 (upsert)
  - 배치/수동 트리거 지원

### 4.2 온보딩 프로세스 (사용자 경험)

가입 후 7단계 셋업 마법사를 통해 CODEF 연동이 자연스럽게 진행됩니다:

1. **회사 정보 입력** — 사업자등록번호, 대표자명, 주소
2. **법인통장 등록** — 은행 선택, 계좌번호, 잔고 입력
3. **홈택스 연동** — 간편인증/공동인증서로 홈택스 연결
4. **금융 데이터 연동 (CODEF)** — 연동할 은행/카드사 선택 + 데이터 수집 동의
5. **직원 등록** — 팀원 이름, 직급, 이메일
6. **첫 프로젝트 생성** — 진행 중인 거래/프로젝트 등록
7. **완료** — 대시보드에서 실시간 경영 현황 확인

**Step 4 상세 (CODEF 연동)**:
- 사용자가 거래 은행과 카드사를 체크박스로 선택
- 금융 데이터 수집 동의 체크
- CODEF Connected ID 자동 발급
- 초기 데이터 동기화 (최근 3개월 거래내역)
- 이후 일 1회 자동 배치 동기화

| **DB 암호화** | Supabase PostgreSQL AES-256 at rest | ✅ 적용 |
| **RLS (Row-Level Security)** | 49개 전체 테이블 회사별 데이터 격리 | ✅ 적용 |
| **JWT 인증** | Supabase Auth + access_token/refresh_token | ✅ 적용 |

| **세션 관리** | JWT 1시간 만료 + refresh_token 7일 | ✅ 적용 |

### 5.3 CODEF 인증정보 관리

- **저장 위치**: `company_settings` 테이블 (Supabase PostgreSQL)
- **접근 제어**: RLS 정책으로 해당 회사의 owner/admin만 접근 가능
- **저장 항목**:
  - `codef_client_id` — CODEF API Client ID
  - `codef_client_secret` — CODEF API Client Secret (암호화)
  - `codef_connected_id` — CODEF Connected ID
  - `codef_connected_at` — 연동 시점
- **전송 보안**: HTTPS (TLS 1.3) 전 구간 암호화
- **로깅**: CODEF API 호출 로그는 개인 금융정보 미포함, 건수/에러만 기록

## 6. 사용자 인터페이스 (스크린샷 설명)

### 6.1 대시보드 — 6대 생존지표

대표가 로그인하면 가장 먼저 보는 화면입니다. CODEF 연동된 은행 잔고가 실시간으로 반영됩니다.

- **현금 잔고**: CODEF 은행 잔액 API → 전 계좌 합산
- **런웨이**: 현금 잔고 / 월 번율 (거래내역 기반 자동 계산)
- **월 번율**: CODEF 거래내역에서 고정 지출 자동 산출
- **미수금**: 세금계산서 매출 - 입금 매칭 차액
- **매출 차트**: 월별 매출 추이 (CODEF 입금 내역 기반)

### 6.2 거래내역 — 자동 수집 + 자동 분류

- **은행 탭**: CODEF → 입출금 내역 일 1회 자동 동기화
- **카드 탭**: CODEF → 법인카드 승인내역 자동 동기화
- **수기 입력 탭**: CODEF 미연동 시 수동 등록 가능
- **자동 분류**: 거래처 패턴 학습 → 15개 카테고리 자동 매핑

### 6.3 설정 — CODEF 연동 관리

설정 페이지의 "외부 연동" 탭에서:
- CODEF Client ID / Secret 입력
- Connected ID 관리
- 연동 상태 확인 + 테스트
- 동기화 수동 트리거
