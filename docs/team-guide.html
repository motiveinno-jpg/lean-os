2 사업본부장 — Day 1 즉시 할 일

아래 서비스에 가입하고, 발급된 키/정보를 대표님 또는 Claude에게 전달해주세요.

API 키, 비밀번호 등 민감 정보는 절대 GitHub이나 공개 채널에 올리지 마세요.
대표님에게 직접(카톡/슬랙 DM) 전달해주세요.
2-1. Vercel 가입 + GitHub 연동 5분
  1. vercel.com 접속
  2. "Sign Up" → "Continue with GitHub" 클릭
  3. GitHub 계정: motiveinno-jpg 으로 로그인
  4. Vercel 대시보드에서 "Import Project" → lean-os 저장소 선택
  5. Framework Preset: Next.js (자동 감지됨)
  6. "Deploy" 클릭
전달: Vercel 프로젝트 URL (예: lean-os-xxxxx.vercel.app)
2-2. Google OAuth 설정 10분
  1. console.cloud.google.com 접속 (회사 Google 계정)
  2. 좌측 메뉴 → "API 및 서비스" → "사용자 인증 정보"
  3. "사용자 인증 정보 만들기" → "OAuth 클라이언트 ID"
  4. 애플리케이션 유형: 웹 애플리케이션
  5. 이름: OwnerView
  6. 승인된 리다이렉트 URI 추가:
    https://njbvdkuvtdtkxyylwngn.supabase.co/auth/v1/callback
전달: 클라이언트 ID + 클라이언트 보안 비밀번호
2-3. Kakao OAuth 설정 10분
  1. developers.kakao.com 접속 → 로그인
  2. "내 애플리케이션" → "애플리케이션 추가하기"
  3. 앱 이름: OwnerView, 사업자명: 모티브이노베이션
  4. 생성 후 → "앱 키" 메뉴에서 REST API 키 복사
  5. "카카오 로그인" → 활성화 설정 ON
  6. "Redirect URI" 추가:
    https://njbvdkuvtdtkxyylwngn.supabase.co/auth/v1/callback
전달: REST API 키
2-4. CODEF API 신청 (은행 연동) 15분 (발급 3~7영업일)
발급에 시간이 걸리므로 가장 먼저 신청해주세요!
  1. codef.io 접속 → 회원가입
  2. "API 사용 신청" 클릭
  3. 필요 서류: 사업자등록증, 대표자 신분증 사본
  4. 용도: "자사 서비스 은행 거래내역 연동"
발급 후 전달: Client ID + Client Secret
2-5. 공공데이터포털 API 신청 5분 (즉시 발급)
  1. data.go.kr 접속 → 회원가입
  2. 검색: "세금계산서" → "국세청 세금계산서 발급 내역 조회 서비스"
  3. "활용신청" 클릭 → 사유: "사내 경영관리 시스템 연동"
전달: API 인증키
2-6. 토스페이먼츠 가맹점 등록 Week 2에 진행
  1. developers.tosspayments.com 접속
  2. "가맹점 등록" → 사업자 정보 입력
  3. 서류: 사업자등록증, 통장사본
  4. 테스트 키는 즉시 발급 → 먼저 전달
  5. 운영 키는 심사 완료 후 (3~5영업일)
전달: 클라이언트 키 + 시크릿 키 (테스트용/운영용 각각)

3 버그/이슈 보고하는 방법

GitHub Issues 사용

github.com/motiveinno-jpg/lean-os/issues 에서 "New Issue" 클릭 후 아래 양식으로 작성해주세요.

[화면이름] 문제 한줄 설명 ## 어디서 발생했나요? - 페이지: (예: 대시보드, 딜 관리, 세금계산서) - URL: (브라우저 주소창의 주소) ## 무엇이 문제인가요? (구체적으로. 예: "매출 합계가 실제와 100만원 차이남") ## 재현 방법 1. OO 페이지에 접속 2. OO 버튼 클릭 3. 문제 발생 ## 스크린샷 (반드시 첨부! 캡처 후 붙여넣기하면 자동 업로드) ## 심각도 - 긴급: 서비스 사용 불가 - 중요: 기능은 되지만 결과가 잘못됨 - 개선: 불편하지만 사용 가능

스크린샷 찍는 법

  • Mac: Cmd + Shift + 4 → 영역 드래그
  • Windows: Win + Shift + S
  • iPhone: 전원 + 볼륨 업 동시 누르기

심각도 기준

이슈 제목 앞에 심각도를 표시해주세요:

  • 긴급 로그인 안 됨, 페이지 안 열림, 데이터 유실
  • 중요 숫자 틀림, 기능 오작동, 저장 안 됨
  • 개선 디자인, 문구, 속도, 편의성

6 자주 묻는 질문

Q: GitHub를 처음 써봐요. 어렵지 않나요?
Issues 탭에서 "New Issue" 버튼만 누르면 됩니다. 글 쓰고 스크린샷 붙이면 끝. 코드를 볼 필요 전혀 없습니다.
Q: 테스트 중 서비스가 안 돼요
개발 중이므로 일시적으로 안 될 수 있습니다. 30분 후 다시 시도해보시고, 계속 안 되면 이슈를 등록해주세요.
Q: API 키를 어디에 전달하나요?
대표님에게 카톡이나 슬랙 DM으로 직접 전달해주세요. 절대 GitHub이나 공개 채널에 올리지 마세요.
Q: 긴급한 문제가 발생했어요
GitHub Issue에 제목 앞에 [긴급] 표시하고, 대표님에게 바로 연락해주세요.
Q: Claude(AI 개발자)와 직접 대화할 수 있나요?
직접 대화는 어렵지만, GitHub Issues에 올리신 내용을 Claude가 확인하고 수정합니다. 이슈에 댓글로 추가 정보를 남기시면 됩니다.
