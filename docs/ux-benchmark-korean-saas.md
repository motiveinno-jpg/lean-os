# Korean SaaS UX 벤치마크 리포트
> OwnerView 개선을 위한 Monday.com / Flex / 모두사인 분석

---

## 1. Monday.com (먼데이) - 프로젝트 관리

### 네비게이션 구조
- **최상위 메뉴: 6개** (Products, AI, Solutions, Resources, Enterprise, Pricing)
- 제품군: Work Management, CRM, Campaigns, Dev, Service (5개 핵심 + 2개 보조 도구)
- **핵심 전략: 역할별 세분화** — 마케팅/PMO/영업/운영/제품/IT/R&D/HR 별로 진입점 제공

### 온보딩
- **무료 플랜**: 2석까지 영구 무료 + Pro 14일 체험
- "Get Started" 단일 CTA → 가입 모달 → 즉시 시작
- 고객 증언: "2주~1개월이면 팀 전체가 익숙해짐"
- **핵심: 멀티스텝 위저드 없이, 바로 보드 생성부터 시작**

### 기능 과부하 해결법
| 전략 | 구현 방식 |
|------|-----------|
| **탭 기반 카테고리** | Projects/Portfolios/Processes/Resources/Goals 5개 탭 |
| **역할별 진입** | 팀 유형 선택 → 해당 기능만 노출 |
| **프로그레시브 디스클로저** | 3단계: 혜택 → 기능 탭 → 유스케이스 |
| **시각적 구분** | 제품별 컬러 아이콘으로 즉시 식별 |

### 빈 상태(Empty State)
- "What would you like to work on?" 인터랙티브 선택기
- 빈 화면 대신 예상 결과물/혜택을 먼저 보여줌

### 모바일
- 반응형 (1440px/1300px/1229px/768px 브레이크포인트)
- 햄버거 메뉴 + 별도 모바일 네비게이션 계층

### OwnerView가 배울 점
1. **역할/상황별 진입점** — "매출 확인하러 왔어요" / "고객 관리하러 왔어요" 같은 맥락별 랜딩
2. **탭 기반 기능 분리** — 대시보드에서 revenue/customers/feedback을 탭으로 전환
3. **빈 상태에 가이드** — 데이터 없을 때 "첫 고객을 등록해보세요" 같은 액션 유도

## 2. Flex (플렉스) - HR/급여

### 네비게이션 구조
- **최상위 메뉴: 4개 카테고리**
  - 구성원 관리 (Employee Management)
  - 성장과 성과 (Growth & Performance)
  - 급여 운영 (Payroll Operations)
  - 비용 관리 (Cost Management)

### 기능 조직화 (핵심 인사이트)
**기능이 아닌 "목적"으로 분류:**

| 카테고리 | 포함 기능 | 프레이밍 |
|----------|-----------|----------|
| 구성원 관리 | 구성원·조직, 근태, 전자계약서, 워크플로우, 인사이트 | "구성원 관리를 위해" |
| 성장과 성과 | 목표, 평가, 1:1, 채용 | "성장과 성과를 위해" |
| 급여 운영 | 급여계산, 연말정산, 복리후생 | "급여 운영을 위해" |
| 비용 관리 | 경비, 법인카드 등 | "비용 관리를 위해" |

**"급여/근태/휴가/증명서"를 기술 용어가 아닌 비즈니스 목표로 재분류한 것이 핵심**

### 온보딩
- 이중 CTA: "무료 체험하기" (셀프서비스) + "도입 문의하기" (컨설팅)
- 소규모는 셀프, 중대형은 상담 → 두 세그먼트 동시 대응

### 기능 과부하 해결법
1. **단계적 공개**: 히어로에 3개 카드만 → 하위 기능은 전용 페이지
2. **목적 기반 그룹핑**: "~를 위해" 프레이밍
3. **고객 증언 우선**: 기능 나열 대신 사회적 증거

### 모바일
- 반응형 (768px/1200px 브레이크포인트)
- iOS/Android 전용 앱 별도 제공

### OwnerView가 배울 점
1. **"목적 기반" 메뉴 구조 도입** — 현재 platform/ 하위가 customers/feedback/revenue/referral/system인데, 이를 "매출 관리를 위해" / "고객 관리를 위해" / "운영 설정" 같은 목적형으로 리프레이밍
2. **4개 이하 최상위 카테고리** — Flex가 복잡한 HR 전체를 4개로 묶은 것처럼, OwnerView도 최상위를 3~4개로 제한
3. **이중 CTA 전략** — "바로 시작" + "도입 상담" 두 트랙

## 3. 모두사인 (Modusign) - 전자서명

### 네비게이션 구조
- **최상위 메뉴: 7개** (솔루션, 이용방식, 가격, 고객사례, 법률/보안, 블로그, 로그인)
- 각 메뉴에 4~5개 하위 옵션

### 서명 플로우 (핵심 UX)
**4단계 선형 프로세스:**
1. 문서 준비 (업로드/설정)
2. 서명 요청 (서명 위치 지정)
3. 전자서명 (수신자 서명)
4. 문서 관리 (실시간 추적)

- **5초 자동 회전 탭**으로 단계를 시각화 (수동 조작 시 멈춤)
- "업로드 → 위치 지정 → 전송" 3액션으로 요약

### 온보딩
- **진입 장벽 최소화:**
  - "1개월 무료 이용" (카드 없이)
  - "서명자 체험하기" → 계정 생성 없이 이메일/카카오톡으로 서명 경험 가능
  - **핵심: 가입 전에 핵심 가치를 체험시킴**

### 가격 구조
| 플랜 | 월 가격 | 연간 가격 |
|------|---------|-----------|
| PERSONAL | ₩9,900 | - |
| TEAM | ₩55,000 | ₩27,900/월 |
| TEAM PRO 360 | ₩88,000 | ₩51,900/월 |
| TEAM PRO 1000 | ₩244,200 | ₩104,900/월 |
| Custom | 협의 | 협의 |

- "가장 많이 찾는" 배지로 의사결정 유도
- 월/연 가격 나란히 표시 + 할인율 강조

### 기능 과부하 해결법
| 전략 | 구현 |
|------|------|
| **3단 계층** | 솔루션(기업용/공공용/AI) → 이용방식(웹/API/온프레미스) → 신뢰(법적/보안) |
| **세그먼트 메시징** | 산업별 9개 + 부서별 11개 유스케이스 |
| **최소 히어로** | 위에 1문장 + 자동회전 탭 1개만 |
| **맥락별 CTA** | 무료체험/상담문의/서명체험 3가지 경로 |

### OwnerView가 배울 점
1. **"체험 먼저" 온보딩** — 가입 전에 핵심 기능(예: 대시보드 미리보기)을 체험시키기
2. **선형 단계 시각화** — 복잡한 프로세스를 3~4단계 탭으로 시각화
3. **"가장 많이 찾는" 뱃지** — 가격/플랜에서 추천 옵션 하이라이트

## OwnerView 즉시 적용 가능한 액션 아이템

### 우선순위 1: 네비게이션 리팩토링
**현재**: customers / feedback / revenue / referral / system (5개 병렬)
**개선안** (Flex 방식 적용):
```
매출 관리    → revenue, referral
고객 관리    → customers, feedback
운영 설정    → system
```
- 5개 → 3개 최상위 카테고리로 축소
- "~를 위해" 프레이밍 적용

### 우선순위 2: 빈 상태 디자인 (Monday 방식)
- 각 페이지에 데이터 없을 때 액션 유도 메시지:
  - Revenue: "첫 매출을 기록해보세요" + CTA 버튼
  - Customers: "고객을 등록하고 관리를 시작하세요"
  - Feedback: "고객 피드백을 수집해보세요"

### 우선순위 3: 온보딩 개선 (모두사인 방식)
- 가입 전 대시보드 미리보기 (샘플 데이터)
- 첫 로그인 시 3단계 가이드: "매출 등록 → 고객 추가 → 피드백 확인"

### 우선순위 4: 프로그레시브 디스클로저
- 대시보드에 핵심 KPI 3개만 표시 (Monday 탭 방식)
- "더보기" 클릭 시 상세 데이터 노출
- 사이드바 메뉴 접기/펼치기

### 우선순위 5: 맥락별 CTA
- 각 화면에 "다음에 할 일" 추천 (모두사인의 맥락별 CTA 패턴)
- 예: Revenue 페이지에서 "이번 달 매출이 좋네요! 추천인 프로그램으로 더 성장해보세요" → Referral 연결

## 핵심 한 줄 요약

> **Monday.com은 "역할별 진입"으로, Flex는 "목적별 그룹핑"으로, 모두사인은 "체험 먼저"로 복잡성을 관리한다. OwnerView는 현재 5개 기능을 병렬 나열하는데, 이를 3개 목적 그룹으로 묶고 + 빈 상태 가이드 + 프로그레시브 디스클로저를 추가하는 것이 가장 시급하다.**
