## 처리 순서 권장
1. **P1 먼저**: R1(급여 정합성), R2(거래내역) — 데이터 신뢰 직결.
2. **R3는 이미지 확보 전 착수 금지** — 직원 첨부 이미지(Notion 직인 카드 🔧 토글) 필요. 사용자에게 이미지 요청.
3. P2(R4·R5·R6·R7) → P3 순. R13+R14는 묶어서, R10+R11(딜)은 묶어서.
4. 도메인 묶음: ops(R1·R6·R7·R8·R13·R14) / finance(R2·R9·R15) / hr(R4·R5·R12) / docs(R3) / growth(R10·R11).
5. 스키마·RLS·다중도메인(R11·R13·R14·R10) = L 티어 → db-architect 선행 + security-reviewer.
