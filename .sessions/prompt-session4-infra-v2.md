# 세션4: 대시보드/인프라 — KAIROS 논쟁 모드

너는 오너뷰(~/lean-os)의 대시보드/인프라/모바일 전담이다.

## ★★★ 핵심 규칙: 페르소나 논쟁

코드를 한 줄이라도 짜기 전에, 반드시 4명의 실사용자가 **논쟁**하는 장면을 작성하라.
자문자답이 아니다. 서로 반박하고, 부족한 점을 지적하고, 대안을 제시하는 실제 논쟁이다.

### 페르소나 4명

**채희웅 (CEO)**: "아침에 폰으로 대시보드 열었을 때 3초 안에 회사 상태가 파악되어야 한다. 어제 매출, 현금 잔고, 미수금, 이번 주 할 일. 이 4개가 첫 화면에 없으면 실패다."

**김회계 (회계담당자)**: "설정에서 알림을 세분화하고 싶다. 세금 마감 알림은 중요하니까 텔레그램으로 보내고, 채팅 알림은 끄고 싶다. 그리고 회계 데이터 내보내기가 설정에 있어야 한다."

**박영업 (사업본부장)**: "처음 가입한 사람이 '이게 뭐야' 하고 나가면 끝이다. 온보딩이 손잡고 이끌어야 한다. 그리고 경쟁사 PT 할 때 대시보드 스크린샷을 보여주는데, 차트가 없으면 임팩트가 없다."

**이신입 (신입 직원, 모바일 유저)**: "지하철에서 폰으로 보는데 사이드바가 화면을 반이나 가린다. 하단 탭 바가 있어야 한다. 그리고 로딩이 느리면 앱이 죽은 줄 알고 새로고침 한다."

## 구체적 과제

### P1: 모바일 하단 네비게이션 → Flex 수준
- 768px 이하에서 하단 탭 바: 홈(대시보드), 딜, 거래, 직원, 더보기
- 각 탭에 아이콘 + 라벨 (활성 탭 primary 색상)
- "더보기" → 전체 메뉴 바텀시트 (나머지 모든 메뉴)
- 상단 헤더: 로고 + 알림 벨 + 프로필 아바타
- 사이드바는 768px 이하에서 숨김

### P2: 대시보드 차트 강화 → Monday.com 수준
- 월별 매출 추이 라인차트 (12개월, SVG)
- 현금흐름 차트 (입금 vs 출금 vs 잔액, 영역 차트)
- 딜 파이프라인 깔때기 (상담→제안→계약→정산, 건수+금액)
- 이번 달 주요 일정 카드 (대출 만기, 세금 마감, 계약 만료, 구독 갱신)

### P3: 설정 알림 세분화
- 알림 종류: 딜 상태변경, 입금 알림, 세금 마감, 구독 만료, 채팅 멘션, 급여일
- 채널: 이메일 / 인앱 / 텔레그램 (각각 ON/OFF 토글)
- 업무시간: 시작~종료 시간 설정, 업무시간 외 방해금지
- 테스트 알림 발송 버튼

### P4: 온보딩 체크리스트 강화
- 실 데이터 입력 유도: 회사정보→첫 거래처→첫 딜→첫 직원→첫 거래내역
- 각 완료 시 체크 애니메이션 (CSS)
- 진행률 퍼센트 바 (대시보드에도 표시)
- "나중에 하기" → 대시보드에 잔여 체크리스트 배너
- 샘플 데이터 원클릭 생성 버튼

### P5: Import Hub 통합 강화
- 거래처/직원/거래내역/세금계산서 타입 선택
- 타입별 엑셀 템플릿 다운로드
- 업로드 → 파싱 → 미리보기 (오류 행 빨간 하이라이트)
- 임포트 히스토리 표시
