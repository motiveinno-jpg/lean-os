프로젝트 개요

OwnerView란? 중소기업 대표가 회사의 돈, 사람, 문서, 거래를 한 곳에서 관리하는 올인원 경영 OS입니다.

목표: 정부사업(네이버광고대행) 참여 기업 약 4,000개사에 무료 1개월 이용권 배포 → 유료 전환

타임라인: 2개월 내 첫 유료 고객 확보 (2026년 6월 초 목표)

현재 상태: 코드 25,000줄 완성, Vercel SSR 배포 완료. 외부 서비스 연동 + QA 단계 진입

접속 주소:

  • https://www.owner-view.com (메인 주소 — 접속 가능)
  • https://lean-os.vercel.app (백업 주소)

CODEF API 사용 신청
긴급 Phase 3 차단

은행/카드 거래내역을 자동으로 불러오는 핵심 API입니다. 발급에 3~7영업일 걸리므로 가장 먼저 신청해야 합니다.

정식 키 발급 후 Supabase 환경변수에 설정 (Claude가 처리)
참고 API 키가 나올 때까지 거래내역은 엑셀 업로드로 대체됩니다. 기능 개발은 테스트 키로 먼저 진행합니다.

서비스 접속 및 회원가입
즉시

https://lean-os.vercel.app 접속
회원가입 (이메일/비밀번호)
로그인 → 대시보드 진입 확인
접속이 안 되거나 에러가 나면 스크린샷 찍어서 공유

2
인증 풀플로우 검증
이번 주

가입부터 로그인까지 전체 흐름이 정상 작동하는지 확인합니다.

  • 이메일 회원가입 → 로그인 → 대시보드 → 로그아웃 → 재로그인
  • 비밀번호 찾기(재설정) 기능 테스트
  • 잘못된 비밀번호 입력 시 에러 메시지 확인
  • 모바일(핸드폰)에서도 동일하게 테스트
3
세금계산서 3-Way 매칭 검증
Phase 2

세금계산서 ↔ 계약서(딜) ↔ 입금내역 자동 대조 기능의 숫자 정확성을 검증합니다.

  • 테스트 딜(프로젝트) 생성 → 계약금액 입력
  • 세금계산서 수동 등록 (공급가액, 세액, 합계금액)
  • 거래내역(입금) 등록
  • 검증 포인트:
  • 공급가액 + 세액 = 합계금액이 맞는지
  • 계약금액과 세금계산서 금액이 일치/불일치 표시가 정확한지
  • 입금액과 세금계산서 금액 매칭이 맞는지
  • 부분 입금 시 잔액 계산이 정확한지
4
급여 자동계산 검증
Phase 2

직원 급여 계산의 정확성을 검증합니다.

  • 테스트 직원 등록 (기본급 300만원 등 설정)
  • 검증 포인트:
  • 국민연금 (4.5%), 건강보험 (3.545%), 장기요양 (건강보험의 12.81%)
  • 고용보험 (0.9%), 소득세, 지방소득세
  • 공제 합계와 실수령액 계산이 정확한지
  • 연봉 기준 / 월급 기준 계산이 맞는지
2026년 요율 기준으로 확인해주세요 보험료율이 매년 변경되므로 현행 요율과 시스템 적용 요율을 대조해주세요.
5
거래내역 자동분류 검증
Phase 2

엑셀로 거래내역을 업로드하면 자동으로 분류되는 기능을 검증합니다.

  • 실제 은행 거래내역 엑셀 파일 준비 (회사 계좌)
  • 대시보드 → 엑셀 업로드 → 자동 분류 결과 확인
  • 수입/지출 구분이 맞는지
  • 거래처명 인식이 정확한지
  • 카테고리 자동 분류가 합리적인지
6
입금 자동매칭 검증
Phase 2~3

새로 구현된 입금 자동매칭 엔진의 정확성을 검증합니다.

  • 수입예정(매출 스케줄)에 예상 금액, 예상 입금일, 송금자명 입력
  • 실제 거래내역 등록 (금액, 날짜, 송금자)
  • 자동매칭 결과 확인: 맞는 건을 맞다고 하는지, 틀린 건을 걸러내는지
  • 85점 이상 자동확정 / 60~84점 승인대기 / 60점 미만 무시 기준이 합리적인지
7
페르소나 B(회계) 시나리오 테스트
