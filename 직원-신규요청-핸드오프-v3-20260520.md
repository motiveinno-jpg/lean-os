- **완료 정의**: 미초대 사용자가 채널 목록·메시지·DM 어느 것도 SELECT 0. 초대받은 사용자만 본인 채널 표시. prod 라이브 스모크(타 직원 계정으로 채널 0행) 첨부.

## 🟠 P1·P2 — 결정·확인 필요 (충돌·재질의 권장)

### B1. 통장 자동이체 연결 방식 전면 재설계
- **출처**: 💰 회계관리 › 통장
- **직원 원문**: "자동이체 연결 방식을 바꿔야할거같음" / "현재: 거래내역에 있는 자동에 체크만 하는 방식 →문제점: 매달 체크눌러야함 매우불편해"
- **라우팅**: finance-agent(bank/transactions) — 60420ce(자동이체 라벨 로직)·5dccbbc(고정비/정기결제 흐름)과 정합
- **해석**: 현재 매달 수동 체크 방식이 비효율. 개선 방향 권장안:
  - **권장 1안**: 처음 한 번 체크하면 같은 패턴(같은 출금처/유사 금액/주기)이 다음 달부터 **자동 인식**돼 자동이체 표시. (rule-based: payee+금액±range+월주기)
  - **2안**: 별도 "자동이체 등록" 화면에서 출금처/금액/주기를 명시 등록 → 매칭 자동.
  - **3안**: CODEF가 제공하는 자동이체 메타가 있으면 그걸 우선 활용.
- **할 것**: 권장 1안 우선 검토 + 직원에게 어느 방향 선호하는지 회신 받아 확정 후 구현(L 티어 가능, 매칭 로직 = pure code).
- **완료 정의**: 한 번 등록·체크하면 다음 달부터 같은 자동이체 거래가 자동 표시. 매달 재체크 0.
- **불확실/재질의 권장**: 직원이 선호하는 방향(자동 인식 vs 명시 등록 vs CODEF 메타 기반) 확인 후 진행 안전.

### A3. 현금영수증 수동등록 제거 ⚠️ BLOCKED 충돌
- **출처**: 💰 회계관리 › 현금영수증 · **직원 원문**: "수동등록은 없애버려요"
- **라우팅**: finance-agent — 현금영수증 라우트
- **⚠️ 충돌**: 메모리 `project_hometax_blocked` — 홈택스/현금영수증 CODEF 자동 sync가 CF-12200/CF-00007 답변 대기로 **BLOCKED** 상태. 수동등록 제거 시 등록 경로 완전 차단 위험.
- **할 것(권장)**: 수동등록 UI를 **숨김(메뉴 제거)** 하되 코드/엔드포인트는 보존(자동 sync 복구 시 즉시 재노출 가능). CODEF 자동 sync 복구되면 수동 완전 제거.
- **하지 말 것**: BLOCKED 영역(`tax-invoices/`, `codef-sync` 의 hometax/cash-receipt 분기) 접근 금지. CF-에러 우회 시도 금지.
- **완료 정의**: 직원에게 수동등록 진입 경로 0(메뉴/버튼 미노출). 자동 sync 복구 전엔 백엔드 보존.
- **불확실/재질의 권장**: 수동등록을 "완전 제거"인지 "당분간 숨김"인지 직원 의도 확인. 자동 sync 복구 일정 = CODEF 답변 대기.

### A4. 거래처원장 — 대폭 재설계 (논의 필요)
- **출처**: 💰 회계관리 › 분석 · **직원 원문**: "💬 거래처원장 — 대폭 수정 필요 — 논의 필요" / "거래처별로 볼수있어야하고 외상매출금/ 미지급금/ 입금/ 출금별로 볼수있어야함" / "결국 잔액이 0이 될수있게 관리하는 장부니까 한눈에 잘 보이게.."
- **라우팅**: ops-agent(reports/분석 허브) + db-architect(거래처원장 RPC/뷰 신규) — **L 티어**, 큰 작업
- **해석**: 표준 회계 거래처원장(transactions ledger by partner) — 거래처별 미수금(외상매출금)·미지급금·입금·출금 → 누적 잔액 표시, 잔액 0 도달 추적.
- **할 것**: (1) 분석 허브에 `거래처원장` 페이지 신설(또는 기존 거래처원장 페이지 재구현). (2) 거래처 선택 드롭다운/검색. (3) 컬럼: 일자/적요/외상매출금/미지급금/입금/출금/누적잔액. (4) 거래처 전체 합계 + 잔액 추이 시각화. (5) 데이터 소스: 거래(통장+카드+세금계산서) ↔ 거래처 매칭. RPC 신규 권장(SECURITY DEFINER + 회사격리).
- **하지 말 것**: 기존 거래처 데이터·매칭 로직 회귀 금지. BLOCKED 영역 접근 금지.
- **완료 정의**: 거래처 선택 → 분개 그리드 + 누적 잔액 + 잔액 0 시각 표시. 직원이 "한눈에 잘 보임" 판단.
- **불확실/재질의 권장**: 데이터 매칭 기준(거래처 자동 매칭 정확도), 미수금/미지급금 정의(세금계산서 미지급 분 인식 시점) 직원/사용자 확인 권장.
